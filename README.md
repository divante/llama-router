# LLM Config Generator with Model Duplication

A lightweight configuration system for llama-server that automatically duplicates models into GPU and CPU variants, enabling dynamic routing based on GPU load.

## Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                    llama-server (single container)           │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Model: qwen3.5:35b                                   │   │
│  │  ┌───────────────────────────────────────────────┐   │   │
│  │  │  :gpu variant (ngl=99)                        │   │   │
│  │  │  - All layers on GPU                          │   │   │
│  │  │  - Fast inference                             │   │   │
│  │  └───────────────────────────────────────────────┘   │   │
│  │  ┌───────────────────────────────────────────────┐   │   │
│  │  │  :cpu variant (ngl=0)                         │   │   │
│  │  │  - All layers on CPU                          │   │   │
│  │  │  - Fallback when GPU busy                     │   │   │
│  │  └───────────────────────────────────────────────┘   │   │
│  └───────────────────────────────────────────────────────┘   │
│  ┌───────────────────────────────────────────────────────┐   │
│  │  Model: DeepSeek-R1-70B                               │   │
│  │  ┌───────────────────────────────────────────────┐   │   │
│  │  │  :cpu variant (ngl=0) only (large model)     │   │   │
│  │  └───────────────────────────────────────────────┘   │   │
│  └───────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
         │
         │ Port 8080 (GPU endpoint) + Port 8081 (CPU endpoint)
         │
         ▼
┌──────────────────────────────────────────────────────────────┐
│              config_gen.py (Configuration Generator)         │
│  - Auto-discovers models from /models volume               │
│  - Generates config.ini with GPU+CPU variants              │
│  - Large models (>20GB) auto-routed to CPU only            │
└──────────────────────────────────────────────────────────────┘
```

## Quick Start

### Generate config.ini

```bash
# Basic usage
uv run python -m llm.llama-router.config_gen

# With custom settings
uv run python -m llm.llama-router.config_gen \
  --models-dir /data/ai-stack/llm/models \
  --output ./llm/llama-server/config.ini \
  --large-model-threshold 20GB \
  --batch-size 512 \
  --ctx-size 32768
```

### Run with Docker Compose

```bash
# Start the llama-server stack
docker compose -f docker-composes/llm-compose.yaml up -d

# View logs
docker compose -f docker-composes/llm-compose.yaml logs -f llama-server
```

### Regenerate config after adding models

```bash
# Generate config locally
uv run python -m llm.llama-router.config_gen \
  --output ./llm/llama-router/config.ini

# Copy to container volume
docker cp ./llm/llama-router/config.ini llama-server:/app/config/config.ini

# Restart to apply changes
docker compose -f docker-composes/llm-compose.yaml restart llama-server
```

## Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| `--models-dir` | `/data/ai-stack/llm/models` | Directory containing GGUF models |
| `--output` | `/app/config/config.ini` | Output config path |
| `--large-model-threshold` | `20GB` | Models larger than this are CPU-only |
| `--batch-size` | `512` | Default batch size |
| `--ctx-size` | `32768` | Default context length |
| `--num-threads` | `8` | Default CPU threads (GPU), `16` (CPU) |
| `--ngl-gpu` | `99` | GPU layers for GPU models |
| `--ngl-cpu` | `0` | GPU layers for CPU models |
| `--routing` | `{}` | JSON model routing override |

### Model Routing Override

Force specific models to GPU or CPU:

```bash
uv run python -m llm.llama-router.config_gen \
  --routing '{"qwen3.5:35b": "gpu", "DeepSeek-R1-70B": "cpu"}'
```

Valid values: `gpu`, `cpu`, `auto` (default)

## Accessing Models

Models are accessible via two ports:

- **Port 8080**: GPU endpoint (primary)
- **Port 8081**: CPU endpoint (fallback)

Example requests:

```bash
# GPU route (port 8080)
curl http://localhost:11430/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3.5:35b:gpu", "messages": [{"role": "user", "content": "Hello"}]}'

# CPU route (port 8081)
curl http://localhost:11431/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3.5:35b:cpu", "messages": [{"role": "user", "content": "Hello"}]}'
```

## Model Naming Convention

Each model gets two endpoints:
- `{model_name}:gpu` - GPU variant
- `{model_name}:cpu` - CPU variant

Large models (>20GB by default) only get the CPU variant.
